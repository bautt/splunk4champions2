# tstats  <img src="/static/app/splunk4champions2/images/fast_search.jpg" width="60%" />

`tstats`` command  performs statistical queries on indexed fields in tsidx files. 
The indexed fields can be from indexed data or accelerated data models.

*This section is based on 2020 Richard Morgans session "TSTATS and PREFIX" and 2021 .conf session "Clara-Fication: More Tstats for Your Buckets" by Clara Merriman and Martin Müller*

## Lets look at this "traditional" search in raw weather data: 
<SplunkSearch earliest="-1y" latest="now" spl={`\`s4c_index_weather\` |timechart span=1h avg(main.temp) avg(visibility) avg(wind.speed) avg(main.humidity) avg(main.pressure) by name limit=0`}></SplunkSearch>

use job inspector and compare it to this "tstats" search: 
<SplunkSearch earliest="-1y" latest="now" spl={`|tstats avg(main.temp) avg(visibility) avg(wind.speed) avg(main.humidity) avg(main.pressure) WHERE \`s4c_index_weather\` earliest=-1y latest=now()  by name _time span=1h| xyseries _time,name,avg*`}></SplunkSearch>


## Simple example: index based tstats search (works with indexed fields)
<SplunkSearch earliest="-30d" latest="now" spl={`| tstats count where index=_* OR index=* by host`}>
</SplunkSearch>

## Filter by indexed fields: 
<SplunkSearch earliest="-30d" latest="now" spl={`| tstats count where \`s4c_index_internal\` source=*metrics.log*`}>
</SplunkSearch>

## Filter by simple indexed terms: 
<SplunkSearch earliest="-30d" latest="now" spl={`| tstats count where \`s4c_index_internal\` source=*metrics.log* typingqueue`}>
</SplunkSearch>

## Filter by complex indexed `TERM`s: 
| tstats count where `s4c_index_internal` source=*metrics.log* TERM(blocked=true)
<SplunkSearch earliest="-30d" latest="now" spl={`| tstats count where \`s4c_index_internal\` source=*metrics.log* TERM(blocked=true)`}>
</SplunkSearch>

## PREFIX(name=) will ad-hoc-interpret the indexed term name=typingqueue as a key-value pair (v8.0+) - even if there is no indexed field "name"
With `PREFIX` indexed fields are not longer a prerequisite for tstats. The extension massively increases the instances where tstats can be used
`PREFIX` allows TERMS to be processed as if they were indexed fields.
<SplunkSearch earliest="-30d" latest="now" spl={`| tstats count where \`s4c_index_internal\` source=*metrics.log* TERM(blocked=true) by PREFIX(name=)`}>
</SplunkSearch>

## “Traditional” search to chart queue sizes:
<SplunkSearch earliest="-30d" latest="now" spl={`\`s4c_index_internal\` source=*metrics.log* group=queue | timechart span=15m max(current_size_kb) by name`}>
</SplunkSearch>

## PREFIX() works for aggregations too!
<SplunkSearch earliest="-30d" latest="now" spl={`| tstats max(PREFIX(current_size_kb=)) as max_size_kb where \`s4c_index_internal\` source=*metrics.log* TERM(group=queue) by _time span=15m PREFIX(name=) | timechart span=15m max(max_size_kb) by "name="`}>
</SplunkSearch>
