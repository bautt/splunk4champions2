## Publishing Dashboards Without Login (New in Splunk 10.0)

One of the most-requested Dashboard Studio features: share a live dashboard with anyone — **no Splunk account required**. The published URL shows real-time data in any browser, on any device.

---

### How to Publish

1. Open a **Dashboard Studio** dashboard (Classic/XML dashboards do not support this feature)
2. Click **Publish** in the top-right toolbar (Splunk Enterprise 10.0+)
3. Set the **refresh interval**: every 5 min / 15 min / 1 hour / manual
4. Set an **expiry**: 7 days, 30 days, or no expiry
5. Copy the generated link and share it

The recipient sees live data without ever logging in to Splunk.

---

### Sharing Method Comparison

| Method | Login Required | Live Data | Interactive | Expiry |
|---|---|---|---|---|
| Splunk Login (classic) | Yes | Yes | Yes | Never |
| PDF Export (scheduled) | No | Snapshot only | No | — |
| Embedded Report (iFrame) | Yes* | Yes | Yes | Never |
| **Publish (10.0+)** | **No** | **Yes** | Partial | Configurable |

*iFrame embedding requires an auth token in embed mode.

---

### Use Cases

- **C-Suite / Management reporting** — share KPI dashboards without provisioning Splunk licenses
- **NOC walls and digital signage** — kiosk displays that auto-refresh
- **External partners / customer dashboards** — share operational views securely
- **Read-only views for business units** — procurement, finance, HR seeing their data
- **Automated report links via email** — scheduled alerts include a live link instead of a PDF

---

### Security Considerations

- The published link contains a **signed token** that authorizes data access
- Searches run under the **context of the user who published** the dashboard — RBAC and data access controls remain in effect
- Recipients cannot modify the dashboard or execute arbitrary searches
- The link can be **revoked** at any time from the Publish settings
- Set an appropriate **expiry** for sensitive data — avoid "no expiry" for dashboards containing PII or confidential metrics

---

### Requirements

- Splunk Enterprise **10.0+** or Splunk Cloud (feature parity varies by version)
- The **Publish** button is enabled by default — no `web.conf` changes required
- The publishing user needs `edit_dashboard` and `schedule_search` capabilities
