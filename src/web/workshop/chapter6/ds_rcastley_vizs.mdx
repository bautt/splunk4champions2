## Canvas-Based Custom Visualizations

<Link to="https://github.com/rcastley/splunk-custom-visualizations" openInNewContext>splunk-custom-visualizations</Link> by Robert Castley is a library of production-ready Splunk visualizations built on the Canvas 2D API. Every example in the repo can be **cloned, built, and installed directly** — no Splunk development experience or Cursor required.

### Ready to Install — No Development Required

All visualizations are pre-built. The only tool you need is `git` and `Node.js` to run the build script (Node.js handles the webpack bundle — it is not needed at runtime inside Splunk).

**Install the full health dashboard:**

```bash
git clone https://github.com/rcastley/splunk-custom-visualizations
cd splunk-custom-visualizations
./splunk_health/build.sh
# → dist/splunk_health.tar.gz
```

**Install any standalone visualization:**

```bash
./build.sh gauge
./build.sh network_topology
./build.sh forwarder_heatmap
# → dist/<name>-1.0.0.tar.gz
```

Then upload via **Apps → Manage Apps → Install app from file**. The visualization immediately appears in Dashboard Studio's visualization picker under your custom category.

### Splunk Health Dashboard

Eight glass-themed visualizations bundled into a single ready-to-use app with a pre-built Dashboard Studio dashboard. Panels auto-refresh every 60 seconds and include `appendpipe` fallbacks so nothing goes blank when data is absent.

| Visualization | SPL data source | What it shows |
|---|---|---|
| **Splunk Status Board** | `\| rest /services/server/health` | Component health tiles — Indexer, Search Head, KV Store, … |
| **License Gauge** | `\| rest /services/licenser/pools` | Daily consumption vs quota, arc gauge |
| **Resource Gauge** | `index=_introspection` | Triple-arc: CPU / Memory / Swap |
| **Indexing Pipeline Flow** | `index=_internal group=queue` | Animated queue fill: parsing → merging → typing → indexing |
| **Forwarder Heatmap** | `index=_internal group=tcpin_connections` | Staleness grid — green (recent) → yellow → red (missing) |
| **Search Activity** | `\| rest /services/server/status` | Slot utilization: scheduled / ad-hoc / other vs max concurrent |
| **Scheduler Health** | `index=_internal sourcetype=scheduler` | Success rate, skip rate, average runtime |
| **Index Storage** | `\| rest /services/data/indexes` | Per-index capacity with hot / warm / cold layers |

All eight vizzes share the same glass-tube design language. Animated effects intensify as conditions worsen. Three colour themes: **default**, **dark**, **neon**.

### Building Your Own with Cursor

The repo includes a `/splunk-viz` skill. Describe what you want and it scaffolds the entire app:

```text
Using /splunk-viz, create a visualization that shows a donut chart
with a center label. It should accept "label" and "value" columns.
```

Cursor generates `visualization_source.js`, `formatter.html`, `visualizations.conf`, webpack config, and `build.sh`. Build and install with:

```bash
./build.sh my_viz_name
$SPLUNK_HOME/bin/splunk install app dist/my_viz_name-1.0.0.tar.gz
```
