## What is SPL2?

SPL2 is Splunk's next-generation search and data preparation language, reaching worldwide general availability on **January 16, 2026**. It is a single, unified language that works across both search and stream contexts — the same syntax you use to query indexed data also powers in-stream processing in Edge Processor and Ingest Processor.

> *"Learn once, use everywhere"* — SPL2 logic written for search can be reused directly in stream processing pipelines.

---

### Availability

| Platform | Version | OS |
|---|---|---|
| Splunk Enterprise | 10.2+ | Unix / Linux only |
| Splunk Cloud Platform | 10.2.0.2511+ | All operating systems |

SPL2 is **not enabled by default** — select it from the language picker in the Search bar, or use the Module Editor (see next section).

---

### Multi-modal Syntax

SPL2 supports two syntax styles in the same language. Use whichever fits your background:

**SPL-like syntax** (familiar to existing Splunk users):
```
FROM s4c_meteo
| WHERE city="Bonn"
| STATS avg(temperature_2m) AS avg_temp BY city
| SORT -avg_temp
| LIMIT 10
```

**SQL-like syntax** (familiar to database users):
```sql
SELECT avg(temperature_2m) AS avg_temp, city
FROM s4c_meteo
WHERE city = "Bonn"
GROUP BY city
ORDER BY avg_temp DESC
LIMIT 10
```

Both styles produce identical results.

### SPL1 → SPL2 Quick Reference

| SPL1 | SPL2 |
|---|---|
| `index=s4c_meteo` | `FROM s4c_meteo` |
| `\| stats count by city` | `\| STATS count() BY city` |
| `\| where x > 5` | `\| WHERE x > 5` |
| `\| head 10` | `\| LIMIT 10` |
| `\| sort -avg_temp` | `\| SORT -avg_temp` |

SPL2 keywords are **case-insensitive** but conventionally written in UPPERCASE. Switching from SPL1 requires **minimal or no rewriting** of existing queries and delivers equivalent performance.

### Where to Use SPL2

| Interface | How |
|---|---|
| **Search & Reporting — Search bar** | Select *SPL2* from the language picker next to the search field |
| **Search & Reporting — Module Editor** | Apps → Search & Reporting → **Modules** tab → New module |
| **Edge Processor / Ingest Processor** | Pipeline editor (SPL2 compatibility profile) |
| **VS Code** | Splunk Extension for Visual Studio Code |
| **REST API** | Programmatic module management and search execution |

---

### Key New Capabilities

| Capability | Description |
|---|---|
| **Modules** | Multi-statement investigation notebooks — write, name, chain, and share multiple searches in one file |
| **Data views** | Admin-controlled, permissioned views over indexes — share subsets of data without granting full index access |
| **Custom functions** | Reusable function declarations shareable across the ecosystem |
| **SPL embedding** | Embed legacy SPL1 searches within an SPL2 pipeline for gradual migration |
| **JSON lambda expressions** | `map`, `reduce`, `filter` operations for modern JSON handling |
| **Custom data types** | Define data types for quality validation at search time |
